basel-crud

BASEL can create and manage CRUDs (Create, Read, Update e Delete) using AngularJS to create views and controllers, based in templates, natives or from themes.

basel-crud create CRUDs based in database tables.

Basic Options

You can use some options in creating CRUDs. These are the basic:

Name Short Long Default value Description
CRUD Name Name of your new CRUD
Table -t --table The database table data on which your CRUD will be based.
Controller -c --controller CRUD name in lowercase, without accents and spaces. Concatenated with "Controller". Controller name
View -v --view CRUD name in lowercase, without accents and spaces. (.html) View name (*.html)
Route -r --route CRUD name in lowercase, without accents and spaces The route (Ex.: persons)
Menu -m --menu 1 Show in main menu (1 - Yes, 0 - No)

CRUD name

The name of your CRUD is the most important part of it. It can be used in the view and controller name, if you dont enter them.

The name is just entered after the basel-crud argument.

$ basel-crud myCrud

Table

You have to enter the name of a database table. Look how to create table here.

Use -t or --table to enter the table name.

$ basel-crud myCrud -t tableName

You can create the table at the same time. Look

If the table dont exists, the app will ask you the columns, the primary key and if you have a auto incremental column.

Controller

To inform the controller name, use -c or --controller

$ basel-crud myCrud -t tableName -c myController

If you dont inform this value, the console will ask you.

Never use the name mainController.

View

To inform the view name, use -v or --view

Remember to always use

$ basel-crud myCrud -t tableName -c myController -v myView.html

If you dont inform this value, the console will ask you.
