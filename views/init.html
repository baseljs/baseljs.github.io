basel-init

To init a new BASEL app, you just have to execute basel-init. The first parameter is the app name, and the name of the folder of the app too.

$ basel-init newApp

In the above example, the repository of the most basic BASEL application is cloned is a folder named newApp.

Options

You can use options in the initialization of your app, look:

Name Short Long Default value Description
Database -d --database database Database name
Cipher -c --cipher false For use encrypted database
Algorithm -a --algorithm aes-256-ctr Algorithm for use encrypted database
Password -p --password Password for use encrypted database
Title -t --title The app name Application Title
Description -e --description BASEL Application Application Description
Url -u --url URL of Git repository

Database

You can inform the name of the database or the file path.

  • If you inform just the name, BASEL will create a file with the informed in the folder called model of your app.
  • And, if you inform the file path, BASEL will connect him and save the reference in basel.json.

$ basel-init myApp -d dbName
//or file path
$ basel-init myApp -d "~/path/file.db"

If the file dont exists, BASEL will create the file. But, if the path dont exists, It will show an error.

Cipher

BASEL uses sqlite-cipher.js to handle encrypted databases. To do it you just have to use -c or --cipher.

$ basel-init myApp -d dbName -c //or --cipher

The console will ask you the algorithm used to encrypt, and the passowrd

Algorithm

To inform the algorithm used to encrypt or decrypt the database use -a or --algorithm.

Look the list of algorithms that you can use.

$ basel-init myApp -d dbName -c -a aes-256-ctr

If you dont inform this value, the console will ask you.

Password

Chose the password to encrypt and decrypt your database file. For inform it type -p or --password

$ basel-init myApp -d dbName -c -p yourPassWord

If you dont inform this value, the console will ask you.

Title

The title that will appear in the window of your application. For inform it use -t or --title

$ basel-init myApp -t "My Title" // or --title

If you dont inform this value, the console will ask you.

Description

The Description of your App in package.json and basel.json. Use with -e or --description

$ basel-init myApp -e "This is my first BASEL app" // or --description

Url

You can create your app based in a repository Git. This base-app needs to have the basic structure of the BASEL. Use with -u or --url

$ basel-init myApp -u "https://github.com/baseljs/basel-app" // or --url
