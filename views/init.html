basel-init

To init a new BASEL app, you just have to execute basel-init. The first parameter is the app name, and the name of the folder of the app too.

$ basel-init newApp

In the above example, the repository of the most basic BASEL application is cloned is a folder named newApp.

Options

You can use options in the initialization of your app, look:

Name Short Long Description
Database -d --database Database name
Cipher -c --cipher For use encrypted database

Database

You can inform the name of the database or the file path.

  • If you inform just the name, BASEL will create a file with the informed in the folder called model of your app.
  • And, if you inform the file path, BASEL will connect him and save the reference in basel.json.

$ basel-init myApp -d dbName
//or file path
$ basel-init myApp -d "~/path/file.db"

If the file dont exists, BASEL will create the file. But, if the path dont exists, It will show an error.

Cipher

BASEL uses sqlite-cipher.js to handle encrypted databases. To do it you just have to use -c or --cipher.

$ basel-init myApp -d dbName -c //or --cipher
